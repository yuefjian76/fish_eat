游戏核心架构方案

三种代码组织方式，各有适用场景

UI 层
菜单、HUD、商店
游戏逻辑层
战斗、技能、移动
核心层
玩家、敌人、碰撞
数据层
存档、配置、成就

A. 模块化分层 ⭐推荐

按职责分层：UI层 → 游戏逻辑层 → 核心层 → 数据层

优点：结构清晰，便于多人协作，测试友好
缺点：前期需要花时间搭建框架

优点：贴合 Phaser 设计，切换场景方便

C. 数据驱动

用 JSON 配置表定义鱼、技能、关卡，代码解析配置生成内容

优点：内容扩展容易，不需要改代码
缺点：调试复杂，需要配套工具
